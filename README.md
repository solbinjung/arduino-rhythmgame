# 🎮 아두이노 I2C LCD 리듬 게임 (Arduino LCD Rhythm Game)

개발 기간: 2023.06
시연 영상: <https://youtu.be/b4S-V8L3bKY>

I2C 16x2 Character LCD의 폰트 표현 한계를 극복하기 위해 **사용자 정의 문자(Custom Character Byte) 기법**을 도입, 제한된 픽셀 안에서 3개 라인의 동적 노트 낙하 애니메이션과 입력 동기화를 구현한 임베디드 피지컬 컴퓨팅 프로젝트입니다.

---

## 🛠️ 하드웨어 구성 및 회로 설계 (Circuit Design)

* **MCU:** Arduino Uno R3
* **Display:** I2C LCD 1602 (LiquidCrystal_I2C 라이브러리 v1.1.2 사용)
* **Input Controller:** 푸시 버튼 3개 (Digital Pin 8, 9, 10)
* **Circuit Stability:** 각 버튼에 **220Ω 풀다운(Pull-down) 저항**을 배치하여 플로팅 현상을 방지하고 디지털 입력 신호의 신뢰성 확보

---

## 💡 핵심 기술적 역량 (Key Implementation)

### 1. 문자형 디스플레이의 그래픽 한계 극복 (Pixel Mapping)
일반적인 텍스트 출력용 LCD에서 리듬 게임의 '노트 하강 연출'을 구현하기 위해, 8x5 픽셀 영역을 바이너리 데이터(`B00000` ~ `B11111`) 단위로 정밀하게 쪼개어 4가지 상태의 커스텀 그래픽을 설계했습니다.
* `play00`: 공백 상태 (이동 흔적 지우기 용도)
* `play01`: 1행 상단에 위치한 가상의 노트 파일
* `play02`: 2행 판정선 위치에 노트를 오버랩한 타격 프레임
* `play03`: 기본 판정선 상태 (2행 하단 고정)

### 2. 가상 프레임 애니메이션 파이프라인 (Frame Animation Loop)
`random(1, 4)`로 결정된 무작위 타겟 라인($j$ 좌표)에서 400ms 주기의 시간 지연(`delay`) 제어를 통해 문자 그리드 내에서 노트가 자연스럽게 떨어지는 시각적 착시(Illusion)를 구현했습니다.
* **1단계:** `(j, 0)` 위치에 `play01` 출력 (노트 등장)
* **2단계:** `(j, 0)`을 공백으로 지우고, `(j, 1)` 위치에 `play02` 출력 (판정선 도달)
* **3단계:** `(j, 1)`을 기본 판정선 `play03`으로 복원

### 3. 실시간 입력 폴링 및 데이터 처리
노트가 판정선 영역(`play02`)에 정확히 도달한 찰나의 프레임 윈도우 안에서, 아두이노의 `digitalRead()` 연산과 무작위 생성 변수 $j$의 조건을 동기화 검사하도록 판정 로직을 설계했습니다.
* 단일 세션당 총 24개의 노트를 공급하며, 성공 시 누적 스코어(`+10`)를 실시간 인터페이스에 반영합니다.
* 게임 종료 시, 조건 분기(`score >= 100`)에 따라 런타임 결과(`Great!` / `Fail..`)를 LCD 화면에 예외 없이 시각화합니다.
