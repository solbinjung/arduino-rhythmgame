# 🎮 아두이노 I2C LCD 리듬 게임 (Arduino LCD Rhythm Game)

> **개발 기간:** 2023.06 (1개월)

> **사용 기술:** Arduino(C++)

> **시연 영상:** <https://youtu.be/b4S-V8L3bKY>

텍스트 출력 전용인 **I2C 16x2 Character LCD의 한계를 극복**하기 위해 **사용자 정의 문자(Custom Character) 기능**을 활용하여 제한된 픽셀 안에서 3개 라인의 동적 노트 낙하 애니메이션과 실시간 입력 판정을 구현한 아두이노 기반 임베디드 프로젝트입니다.
---

## 🛠️ 하드웨어 구성 및 회로 설계

* **MCU:** Arduino Uno R3
* **Display:** I2C LCD 1602 (LiquidCrystal_I2C 라이브러리)
* **입력 장치:** 푸시 버튼 3개 (Digital Pin 8, 9, 10)
* **회로 안정성:** 각 버튼에 **220Ω 풀다운(Pull-down) 저항**을 배치하여 플로팅 현상을 방지하고 디지털 입력 신호의 신뢰성 확보

---

## 💡 핵심 구현 및 기술적 문제 해결

### 1. Custom Character를 활용한 그래픽 표현 한계 극복
문자만 출력할 수 있는 LCD 환경에서 리듬 게임의 '노트 하강 연출'을 구현하기 위해 8x5 픽셀 영역을 바이너리 데이터(`B00000` ~ `B11111`) 단위로 제어하여 4가지 상태의 커스텀 그래픽 프레임을 직접 설계했습니다.
* `play00`(공백): 이동 흔적을 지우기 위한 빈 프레임
* `play01`(노트 생성): 1행 상단에 위치한 가상 노트 프레임
* `play02`(노트 도달): 2행 판정선 위치에 노트를 오버랩한 타격 프레임
* `play03`(기본 판정선): 2행 하단에 고정된 기본 판정선 프레임

### 2. 주기적 프레임 제어를 통한 애니메이션 루프 구현
`random(1, 4)` 함수로 무작위 타겟 라인(1~3번 라인)을 결정하고 400ms 주기의 프레임 제어를 통해 노트가 판정선으로 자연스럽게 하강하는 상태 변화 루프를 구현했습니다.
* **1단계 (노트 등장):** 해당 라인의 1행 `(j, 0)` 위치에 `play01` 출력
* **2단계 (판정선 도달):** 1행을 지우고 `(j, 1)` 위치에 `play02` 출력
* **3단계 (프레임 복원):** 입력 판정 후에 2행을 다시 기본 판정선`play03`상태로 복원

### 3. 실시간 입력 폴링 및 판정 로직 초기화
노트가 판정선 영역(`play02`)에 정확히 도달한 찰나의 프레임 타이밍(Window) 내에서 플레이어의 버튼 입력을 정확히 감지하고 처리하도록 설계했습니다.
* 'digitalRead()'를 통해 입력된 버튼 핀 번호와 무작위로 생성된 노트 라인 조건이 일치하는지 실시간으로 검사합니다.
* 세션당 총 24개의 노트를 무작위로 생성하며 판정 성공 시에 누적 스코어를 누적(`+10`)하여 실시간으로 반영합니다.
* 게임 종료 시, 조건문 분기(`score >= 100`)를 통해 최종 결과(`Great!` / `Fail..`)를 LCD 화면에 최종 시각화하여 예외 상황 없이 세션을 마무리합니다.
